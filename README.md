# 🤖 KworkBot

Telegram-бот для автоматического мониторинга фриланс-заказов на [kwork.ru](https://kwork.ru).  
Фильтрует заказы, подходящие для «вайб-кодинга», с помощью ИИ (Groq API) и уведомляет вас в Telegram.

---

## ⚡ Возможности

- 🔍 Парсинг 4 категорий kwork.ru каждые 15 минут
- 🤖 AI-фильтрация заказов через Groq API (llama-3.3-70b)
- 📬 Telegram-уведомления с кнопками
- ✍️ Генерация профессионального отклика на заказ одной кнопкой
- 📊 Статистика и управление прямо из бота
- 🛡 Защита от дублирования заказов (SQLite)

## 🚀 Быстрый старт

### 1. Клонировать / скачать проект

```bash
git clone <repo_url>
cd kworkbot
```

### 2. Установить зависимости

```bash
pip install -r requirements.txt
```

### 3. Настроить окружение

```bash
cp .env.example .env
```

Открыть `.env` и заполнить:

| Переменная              | Описание                                               |
|-------------------------|--------------------------------------------------------|
| `BOT_TOKEN`             | Токен бота от [@BotFather](https://t.me/BotFather)     |
| `GROQ_API_KEY`          | API-ключ с [console.groq.com](https://console.groq.com)|
| `GROQ_MODEL`            | Модель Groq (по умолчанию `llama-3.3-70b-versatile`)  |
| `YOUR_TELEGRAM_ID`      | Ваш числовой Telegram ID (узнать у [@userinfobot](https://t.me/userinfobot)) |
| `PARSE_INTERVAL_MINUTES`| Интервал парсинга в минутах (по умолчанию `15`)        |

### 4. Запустить бота

```bash
python bot.py
```

## ⚠️ Важные ограничения

- **Парсинг kwork.ru** — сайт может блокировать запросы. В коде реализованы случайные задержки и ротация User-Agent. Для боевого использования рассмотрите прокси.
- **Groq API лимиты** — на бесплатном тарифе есть ограничения по RPM. При большом потоке заказов встроены задержки между запросами к AI.
- **Структура HTML kwork.ru** — разметка сайта может меняться. Если парсер перестал находить заказы — нужно адаптировать селекторы в `parse_cards()`.
